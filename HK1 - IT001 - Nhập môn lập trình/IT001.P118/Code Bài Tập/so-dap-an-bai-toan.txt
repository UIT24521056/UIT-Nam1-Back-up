Đề: Bạn Bình vừa gặp một đề Nhập môn lập trình rất dễ như sau: Cho số nguyên dương N, tìm 4 số nguyên dương a, b, c, d sao cho ab+cd=N. Sở dĩ bài này dễ vì có thể có rất nhiều đáp án.
    Ví dụ 2*3 + 2*2 = 10 mà 2*1 + 2*4 thì cũng là 10. 
    Đề dễ thì để Bình làm, còn công việc của các bạn là đếm xem với một số nguyên dương N như vậy thì có tổng cộng bao nhiêu đáp án thỏa mãn yêu cầu?

Testcase: 
	+Input: 4 | Output: 8 | Giải thích: 8 đáp án có thể có đó là 1*1+1*3=4; 1*1+3*1=4; 1*2+1*2=4; 1*2+2*1=4; 1*3+1*1=4; 2*1+1*2=4; 2*1+2*1=4; 3*1+1*1=4
	+Input: 292 | Output: 10886 |
	+Input: 19876 | Output: 2219958 |

Idea: -Phép cộng, phép nhân có tính chất giao hoán => số kết quả luôn chẵn => tìm một nửa kết quả rồi nhân 2.
      +Tìm số phần tử của phép cộng:
        *Giả sử số chẵn 10, ta có 1+9; 2+8; 3+7; 4+6; 5+5; 6+4; 7+3; 8+2; 9+1 => có 9 cách cộng.
        *Giả sử số lẻ 9, ta có 1+8; 2+7; 3+6; 4+5; 5+4; 6+3; 7+2; 8+1 => có 8 cách cộng.
	  => số cách cộng = số đó -1; duyệt các phần tử của phép cộng từ 1=>số đó-1.
      +Tìm số phần tử của phép nhân:
        *Cách nhân của 1 số = duyệt từ 1 đến <bé hơn> căn của số đó , xem số nào chia hết, đem tổng kết quả nhân 2 (nếu căn là số nguyên thì +1) sẽ tìm được số cách nhân.
[[xét ví dụ : 6 => 1+5; 2+4; 3+3; 4+2; 5+1; các phép nhân có thể có : 1*1; 2*1; 1*2; 3*1; 1*3; 4*1; 1*4; 2*2; 5*1; 1*5.]]
Duyệt từ 1->6-1 :
	+tổng cách chọn =tổng cất cả các cách chọn của số hạng 1 * cách chọn của số hạng 2.

Code:
------------------------------------

#include <iostream>
using namespace std;
int socachnhan(double n){
    int k=n;
    int kq=0;
    for(int i=1;i*i<=n;i++){
        if(k%i==0){
            if(i*i==n){
                kq+=1;
            }else{
                kq+=2;
            }
        }
    }
    return kq;
}
void kq(double a){
    int cachnhan=0;
    for(int ts1=1;ts1<a;ts1++){
        int ts2=a-ts1;
        cachnhan+=socachnhan(ts1)*socachnhan(ts2);
    }
    cout<<cachnhan<<endl;
}
int main(){
    double a;
    cin>>a;
    kq(a);
    return 0;
}
